# Week 2 – Penetration Testing Techniques

## Overview

This week focuses on learning the basic concepts and workflow of penetration testing. The goal is to understand how ethical hackers identify and test vulnerabilities in systems using structured methodologies.

## Penetration Testing Phases

1. **Reconnaissance** – Collect information about the target using OSINT tools (e.g., Shodan).
2. **Scanning** – Identify open ports, services, and vulnerabilities (e.g., Nmap, Nessus).
3. **Exploitation** – Use vulnerabilities to gain system access (e.g., Metasploit).
4. **Post-Exploitation** – Escalate privileges and assess system impact.
5. **Reporting** – Document vulnerabilities, risks, and remediation steps.

## Methodologies Studied

* **PTES** – Provides a structured penetration testing process.
* **OWASP WSTG** – Guide for testing web application security.

## Ethics

Penetration testing must be conducted with proper authorization and within the defined scope to ensure responsible and legal security testing.

## Learning Outcome

Gained knowledge of the penetration testing lifecycle, methodologies, and commonly used tools for ethical hacking.
